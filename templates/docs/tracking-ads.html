Tracking Overview

The Tracking API provides analytics and visitor tracking functionality for the e-commerce platform. It tracks user behavior, page visits, and provides analytics data for vendors.

📋 Tracking Headers

All requests should include the following headers for proper tracking:

X-COMERCIFY-VISITOR: <tracker_id> Origin: <client_domain> User-Agent: <browser_info>

Tracking Model

Each visit is tracked with the following information:

{ "tracker_id": "string (UUID)", "client_url": "string", "client_path": "string", "api_path": "string", "browser": "string", "sub_domain": number (website ID), "ip_address": "string", "logged_in": boolean }

Tracking Process

  1. When a user first visits the site, they are assigned a unique tracker_id
  2. All subsequent requests include this ID in the X-COMERCIFY-VISITOR header
  3. The tracking middleware logs each API request
  4. Visits are associated with users when they are logged in

Visit Analytics

GET /tracking/visits/ Vendor

Get visit statistics for a vendor's store

Response

{ "type": "success", "data": { "total_visits": 1234, "unique_visitors": 567, "logged_in_visits": 234, "visit_by_page": { "/products/": 456, "/categories/": 234, "/": 544 } } }

User Behavior Analytics

GET /tracking/behavior/ Vendor

Get user behavior analytics

Response

{ "type": "success", "data": { "most_viewed_products": [ { "product_id": 42, "views": 1234 }, { "product_id": 15, "views": 987 } ], "conversion_rate": 3.5, "average_session_duration": 180 } }

Advertising API

GET /ads/

Get active advertisements

Query Parameters (optional)

ParameterTypeDescription
pageintegerPage number for pagination
limitintegerNumber of ads per page

Response

{ "type": "success", "data": [ { "id": 1, "title": "Summer Sale", "image": "https://cdn.comersify.com/ads/summer-sale.jpg", "link": "https://store.com/summer-sale", "is_active": true, "start_date": "2024-06-01", "end_date": "2024-08-31" } ] }

Response Fields

FieldTypeDescription
idintegerUnique ad identifier
titlestringAd title or headline
imagestringURL to ad image
linkstringURL to redirect when clicked
is_activebooleanWhether ad is currently active
start_datedatetimeCampaign start date (optional)
end_datedatetimeCampaign end date (optional)

🔒 Privacy Considerations

  • IP addresses are stored but not exposed through the API
  • User tracking IDs are randomly generated UUIDs
  • Personal information is only associated when users are logged in
  • Tracking data is scoped to individual vendor stores

💡 Best Practices

For Ads

  • Cache ads locally - Don't fetch on every page load
  • Handle missing images - Show fallback for broken images
  • Track impressions - Log when ads are displayed
  • Track clicks - Log when ads are clicked
  • Respect timing - Don't show expired campaigns
