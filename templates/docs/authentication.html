Sign Up

POST /signup/

Create a new customer account

Request Body

{ "firstName": "John", "lastName": "Doe", "email": "john@example.com", "phoneNumber": "+1234567890", "password": "securepassword", "passwordConfermation": "securepassword" }

Parameters

FieldTypeDescription
firstNamestringUser's first name
lastNamestringUser's last name
emailstringValid email address
phoneNumberstringContact phone number
passwordstringAccount password (min 8 chars)
passwordConfermationstringMust match password

Sign Up with Provider

POST /signup/{provider}/

Sign up using a third-party provider (Google)

Request Body

{ "token": "google-oauth-token", "userType": "CUSTOMER" }

Supported Providers

ProviderDescription
googleGoogle OAuth sign up

Login

POST /login/

Login with email and password

Request Body

{ "username": "john@example.com", "password": "securepassword" }

Response

{ "type": "success", "name": "John Doe", "image": "https://..." }

Refresh Token

POST /refresh-token/

Get a new access token using refresh token

Request Body

{ "refresh": "your-refresh-token" }

Response

{ "type": "success", "access": "new-access-token", "refresh": "new-refresh-token" }

User Settings

GET /settings/

Get current user's settings (Auth required)

Response

{ "type": "success", "data": { "first_name": "John", "last_name": "Doe", "email": "john@example.com", "phone_number": "+1234567890", "image": "https://..." } }
POST /settings/update/

Update user settings (Auth required)

Request Body (all optional)

{ "firstName": "Jane", "lastName": "Smith", "email": "jane@example.com", "phoneNumber": "+0987654321", "oldPassword": "current-password", "password": "new-password", "passwordConfermation": "new-password", "file": "profile-image" }

Store Management

GET /store/{id}/

Get store details by ID

GET /stores/top/

Get list of top-performing stores

GET /customers/

Get list of customers (Vendor/Admin only)

GET /reviews/

Get recent app reviews

Password Management

POST /reset-password/

Request password reset email

Request Body

{ "email": "john@example.com" }

Error Responses

{ "type": "error", "message": "Error description" }

Authentication

Most endpoints require JWT authentication:

Authorization: Bearer <access_token>

Public endpoints: Sign Up, Login, Reset Password
